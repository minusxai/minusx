## How dashboard parameters work

When you add questions with [parameters](/docs/questions/parameters) to a dashboard, MinusX automatically merges parameters that share the **same name and type** into a single dashboard-level filter.

For example, if three questions each have a `:start_date` parameter (type: date), the dashboard displays one date picker that controls all three queries.

## Parameter behavior

- Parameters are **merged by name + type** — both must match
- Parameter types are **locked** in dashboard view (editable only in the question editor)
- Changing a dashboard filter re-executes all affected questions
- Each question retains its own default parameter values

## Adding parameters

To add a filterable parameter to your dashboard:

1. Open a question and add a parameter to its SQL (e.g., `:region`)
2. Save the question
3. Add it to the dashboard
4. The parameter appears as a dashboard-level filter automatically
