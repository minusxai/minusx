## Supported databases

| Database | Type | Notes |
|----------|------|-------|
| **DuckDB** | Local file | Default for local development. Fast, embedded analytics. |
| **PostgreSQL** | Remote | Production databases, data warehouses |
| **BigQuery** | Cloud | Google Cloud data warehouse |
| **Athena** | Cloud | AWS Athena over S3 |
| **ClickHouse** | Remote | Column-oriented OLAP databases |
| **SQLite** | Local file | Single-file databases |
| **CSV Files** | File upload | Upload CSV files — automatically converted to a queryable DuckDB database |
| **Excel (XLSX)** | File upload | Upload spreadsheets — converted the same way as CSV |
| **Google Sheets** | Cloud | Import public Google Sheets as queryable tables |

## Configuring a connection

Database connections are managed in **Settings → Databases** (admin-only), like LLM providers:

1. Open **Settings → Databases**
2. Pick the database type and name the connection
3. Enter connection details (host, port, database, credentials)
4. Save

Credentials never sit in the config document — on save they are extracted into a server-only secrets store and replaced with references, so no read path (API, agent, export) ever sees a raw credential.

<Callout type="info">
CSV, Excel and Google Sheets are **not** connections — they are [datasets](/docs/data/datasets): files any editor can add to their own folder and query immediately.
</Callout>

## How connections are used

- **Questions** execute SQL against a specific connection
- **Knowledge Base** loads schema information from connections for table whitelisting
- **AI Agent** uses connection schemas to write accurate SQL

Each question is tied to a connection. When you create a new question, it uses the default connection unless you specify otherwise.
