The Knowledge Base is how you teach MinusX about your data. The more context you provide, the better the AI assistant performs — writing more accurate queries, using the right tables, and understanding your business logic.

The Knowledge Base has three components:

## Table Context

Tell MinusX which tables and columns are relevant by whitelisting specific schemas. This prevents the AI from using irrelevant tables and reduces noise.

[Learn more about table context →](/docs/knowledge-base/table-context)

## Text Context

Provide free-form documentation about your data — metric definitions, naming conventions, business rules, and anything else that helps the AI understand your domain.

[Learn more about text context →](/docs/knowledge-base/text-context)

## Evals

Measure and improve AI quality over time by creating evaluation sets — pairs of questions and expected SQL queries that test whether the AI is getting it right.

[Learn more about evals →](/docs/knowledge-base/evals)
