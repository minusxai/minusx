## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) (the from-source path also needs Docker Compose)
- An [Anthropic API key](https://console.anthropic.com/) — or any [supported LLM provider](/docs/self-hosting/llm-providers)

---

## Option A — install script (recommended)

Pulls the prebuilt image from GitHub Container Registry and, **while it downloads**, walks you through setup in the terminal — workspace and admin account, AI model provider (with a real connection test), and optionally your first database connection. When the app comes up, everything is already configured: just log in.

```bash
curl -fsSL https://minusx.ai/install.sh | bash
```

Prefer to read it before you run it: [minusx.ai/install.sh](https://minusx.ai/install.sh) (it's [in the repo](https://github.com/minusxai/minusx/blob/main/install.sh)). The script checks Docker, creates a `minusx/` directory, generates `NEXTAUTH_SECRET` into `frontend/.env`, pulls the image, validates your answers against the app's own code (a one-token LLM call and a real database connector test, run inside the image), creates the workspace, and starts MinusX on the first free port from 3000.

Every setup question is skippable — anything you skip is handled by the in-app setup wizard on first login. Running non-interactively (no terminal) skips all questions.

Re-running the same command later **upgrades in place** — an existing workspace is detected, the questions are skipped, and your data in `minusx/data/` survives restarts and upgrades. 

MinusX will be available at [http://localhost:3000](http://localhost:3000).

## Option B — from source

Builds the image locally instead of pulling the prebuilt one.

### 1. Clone the repo

```bash
git clone https://github.com/minusxai/minusx.git
cd minusx
```

### 2. Configure environment variables

MinusX is a single Next.js app — the AI agent orchestrator runs in-process, so
all configuration lives in `frontend/.env` (there is no separate backend).

```bash
cp frontend/.env.example frontend/.env
```

Edit `frontend/.env` — add a generated NextAuth secret:

```bash
# Generate with: openssl rand -base64 32
NEXTAUTH_SECRET=<generated-secret-here>
```

The LLM provider is configured **in the app** after first launch — the setup
wizard's AI Models step (or Settings → Models) stores providers and API keys in
the workspace; see [LLM Providers](/docs/self-hosting/llm-providers).

### 3. Download sample data

Download the **mxfood** demo dataset (~140MB) — a food delivery analytics database:

```bash
mkdir -p data
curl -L -o data/mxfood.duckdb https://github.com/minusxai/sample_datasets/releases/download/v1.0/mxfood.duckdb
```

### 4. Start MinusX

```bash
docker compose up -d
```

MinusX will be available at [http://localhost:3000](http://localhost:3000).

<Callout type="info">
For local development without Docker, see the [development setup guide](https://github.com/minusxai/minusx/blob/main/LOCAL_DEV.md).
</Callout>
