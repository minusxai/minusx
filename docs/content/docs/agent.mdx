## What it can do

- **Write SQL** from natural language descriptions
- **Edit queries** — add filters, change aggregations, join tables
- **Configure charts** — choose chart types, set axes, customize appearance
- **Build dashboards** — add cards, arrange layout, create new questions
- **Explain** — describe what a query does or why results look a certain way

---

## Context awareness

The agent always knows about:

- Your **Knowledge Base** — whitelisted tables and text context
- The **current page** — what query, chart, or dashboard you're looking at
- The **conversation history** — previous messages in the current session

This means you can have natural follow-up conversations: "now break that down by region" or "make it a bar chart instead."

---

## How it works

1. **You send a message** — a question, request, or instruction
2. **The agent plans** — decides what tools to use (write SQL, modify chart, etc.)
3. **Tools execute** — actions happen across the stack (query data, update UI)
4. **Results return** — you see the outcome directly in the page
5. **You iterate** — follow up with refinements or new questions

Every action produces visible, editable artifacts. The agent shows you what it did, and you can undo or modify the result.

<Callout type="info">
The agent reads your Knowledge Base before every query — maintaining good context is the single most impactful thing you can do to improve AI quality.
</Callout>
