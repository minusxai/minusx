The MinusX agent is not a chatbot bolted onto a BI tool. It's a first-class part of the platform — present on every page, aware of your current context, and capable of taking real actions.

---

## Where it lives

The agent is available everywhere you work in MinusX:

- **Explore** — full-page chat for ad-hoc questions. Ask anything, get SQL + results instantly.
- **Questions** — sidebar chat that sees your current query, parameters, and results. Say "add a date filter" and it modifies the SQL directly.
- **Dashboards** — sidebar chat that understands your layout and questions. Say "add a revenue chart" and it creates the question, configures the viz, and places the card.

---

## What it can do

The agent operates through a defined set of tools — it doesn't have unrestricted access. Every action produces a visible, editable artifact:

| Action | What happens |
|--------|-------------|
| Write SQL | Query appears in the editor — you can review and edit |
| Configure chart | Visualization updates in real time |
| Add dashboard card | Card appears in the grid — drag to reposition |
| Search schema | Looks up tables and columns from your connections |

Nothing is hidden. If the agent writes a 50-line query, you see all 50 lines.

---

## Context awareness

The agent reads three layers of context before every action:

1. **Knowledge Base** — whitelisted tables, metric definitions, business rules
2. **Current page** — the SQL, chart, or dashboard you're looking at right now
3. **Conversation history** — what you've discussed in this session

This is why it understands "now break that down by region" — it knows what "that" refers to.

---

## Trainability

The agent gets better as you invest in context:

- **Add table context** → the agent uses the right tables
- **Write text context** → the agent understands your metrics and conventions
- **Run evals** → you measure accuracy and find gaps
- **Iterate** → improve context based on eval results

This is the core loop. The agent is only as good as the context you give it.

<Callout type="info">
The agent reads your Knowledge Base before every query. Maintaining good context is the single most impactful thing you can do for accuracy.
</Callout>
