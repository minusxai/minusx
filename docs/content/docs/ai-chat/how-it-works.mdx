MinusX uses a multi-tier orchestration architecture to power its AI assistant. Understanding how it works helps you get better results.

## The conversation loop

When you send a message to the AI:

1. Your message is added to the conversation log along with the current page context
2. The AI analyzes your request and decides which tools to use
3. Tools execute across different tiers (writing SQL, fetching data, updating UI)
4. Results feed back to the AI for the next step
5. The loop continues until the task is complete

## Tool-based actions

The AI doesn't just generate text — it takes actions through tools:

- **Query tools** — execute SQL, search schemas, load files
- **UI tools** — modify the current question, update chart settings, edit dashboard layout
- **Data tools** — fetch query results, inspect column types

Each tool action produces a visible result in your workspace. Nothing happens behind the scenes.

## Knowledge integration

Before generating any query, the AI loads:

- Your **table context** — which tables and columns are available
- Your **text context** — business rules, naming conventions, metric definitions
- The **current page state** — existing SQL, parameters, chart configuration

This combined context is what makes the AI accurate for your specific data.

## Conversation history

Each page maintains its own conversation history. You can:

- Continue a previous conversation where you left off
- Reference earlier messages and results
- Build on previous queries iteratively
